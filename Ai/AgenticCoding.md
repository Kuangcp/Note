# Agentic Coding

打开思路：
1. 旧项目 技术架构，业务架构，数据流程 梳理，历史业务功能修改方案。
1. 新需求技术选型，技术方案输出，单元测试等验证。


Vibe Coding（氛围编程）与 Spec Coding（规范驱动编程）是 AI 辅助开发时代的两种核心工作流。它们最本质的区别在于：Vibe Coding 是“直觉探索派”，依赖即兴对话和反复试错，适合快速构建原型；而 Spec Coding 是“传统工程派”，主张“规范先行，代码自动”，通过结构化文档严格约束 AI，适合中大型和需要长期维护的复杂工程。

> 核心差异对比表

| 维度 | Vibe Coding（氛围式编码） | Spec Coding（规范驱动编码） |
|---|---|---|
| 开发哲学 | 边想边做，凭直觉、感觉走 | 先想清楚再动手，方案自洽 |
| 核心媒介 | 口语化的对话（Prompt） | 结构化的规格文档（.md, Schema） |
| 开发者角色 | 体验官、纠错侦探、灵感提供者 | 架构师、蓝图设计师、质量评审员 |
| AI 扮演角色 | 独立盲盒开发者（猜你的意图） | 严格按图施工的“施工队” |
| 代码首次通过率 | 极不稳定，容易顾此失彼 | 极高（目标通常在 95% 以上） |
| 交付物 | 只有代码本身（文档通常滞后或缺失） | “规范文档 + 自动化测试 + 代码” |
